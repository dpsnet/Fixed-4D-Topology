## How to Contribute

### Reporting Issues

If you find errors in the paper, typos, or have suggestions:

1. **Check existing issues** to avoid duplicates
2. **Open a new issue** with:
   - Clear title describing the problem
   - Location (chapter, section, equation number)
   - Description of the issue
   - Suggested fix (if applicable)

### Improving Documentation

Documentation improvements are always welcome:
- Fixing typos
- Clarifying explanations
- Adding examples
- Improving figure captions

### Code Contributions

For figure generation scripts or numerical validation code:

1. **Fork the repository**
2. **Create a branch**: `git checkout -b feature/your-feature`
3. **Make your changes**
4. **Test thoroughly**
5. **Submit a pull request**

## Contribution Guidelines

### For Scientific Content

1. **Mathematical rigor**: Verify or improve L1 strictness claims (currently pending verification)
2. **Physical plausibility**: Ensure physical interpretations make sense
3. **Testability**: Prefer predictions that can be tested experimentally
4. **Documentation**: Explain the motivation and implications
5. **Transparency**: Acknowledge AI-generated vs. human-verified content

### For Code

1. **Style**: Follow PEP 8 for Python
2. **Documentation**: Add docstrings and comments
3. **Testing**: Include tests where applicable
4. **Dependencies**: Minimize external dependencies

### For Documentation

1. **Clarity**: Write for a broad physics audience
2. **Accuracy**: Verify all technical statements
3. **Completeness**: Include sufficient context
4. **Consistency**: Use established terminology

## Pull Request Process

1. **Update README**: If adding new features
2. **Update documentation**: Reflect any API or conceptual changes
3. **Add tests**: For code contributions
4. **Ensure compatibility**: LaTeX compiles without errors
5. **Request review**: Tag maintainers for review
