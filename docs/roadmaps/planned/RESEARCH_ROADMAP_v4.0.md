## 重建方法论

### 三阶段审查流程

```
阶段1: 概念审查 (Concept Screening)
├── 输入: M-或P-声明
├── 评估: 是否符合物理直觉？
├── 输出: 通过/拒绝/修改
└── 时间: 1-2周/概念

阶段2: 数学形式化 (Mathematical Formalization)
├── 输入: 通过的概念
├── 任务: 用T1-T4语言重表述
├── 输出: 数学猜想
└── 时间: 2-4周/概念

阶段3: 严格证明 (Rigorous Proof)
├── 输入: 数学猜想
├── 任务: 存在唯一性证明
├── 输出: L1定理 或 L2渐进结果
└── 时间: 1-3月/定理
```

### 严格性评估标准

| 等级 | 要求 | 标记 |
|------|------|------|
| **L1** | 完整证明，无假设 | ✅ 严格定理 |
| **L2** | 渐进/近似，误差可控 | ⚠️ 渐进结果 |
| **L3** | 启发式，待证明 | ❓ 研究假设 |
| **拒绝** | 无法形式化或与公理矛盾 | ❌ 舍弃 |

## 质量控制机制

### 审查检查清单

每个重建的定理必须通过：

- [ ] **公理回溯**: 能否从A1-A9推导？
- [ ] **存在性**: 解是否存在？
- [ ] **唯一性**: 解是否唯一？
- [ ] **渐近行为**: 与已知极限(GR/QFT)是否匹配？
- [ ] **数值验证**: 计算结果是否合理？
- [ ] **实验可检验**: 是否有可观测预言？
- [ ] **诚实标记**: 严格性等级是否准确？

### 文档要求

每个重建的理论必须包含：

1. **概念来源**: 来自M-或P-的哪个声明
2. **原声明分析**: 为什么不严格？问题在哪？
3. **重建过程**: 如何用T1-T4重新表述
4. **严格证明**: 完整数学推导
5. **严格性评估**: L1/L2/L3标记
6. **开放问题**: 哪些部分仍需完善
