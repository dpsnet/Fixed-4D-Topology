### 咨询策略

#### 1. 分阶段接触

```
第1-2周: 准备阶段
├── 整理L1证明文档
├── 撰写技术摘要
├── 准备专家特定问题
└── 制作联系材料

第3-4周: 初步接触
├── 发送介绍邮件 (所有专家)
├── 安排会议时间
└── 跟进未回复的专家

第5-6周: 咨询会议
├── 进行技术咨询
├── 收集书面反馈
└── 记录关键建议
```

#### 2. 咨询材料包

每个专家将收到:

1. **技术摘要** (3-5页)
   - 研究背景和动机
   - 核心定理陈述
   - 主要贡献概述

2. **专家特定问题**
   - 与该专家领域相关的问题
   - 需要特别验证的论证
   - 开放性问题

3. **完整证明文档** (可选)
   - L1严格证明文档
   - 数值验证结果
   - 技术附录

#### 3. 反馈整合

```python
反馈整合流程:
1. 收集所有专家反馈
2. 分类整理意见:
   - 证明结构建议
   - 技术细节修正
   - 引用建议
   - 开放问题
3. 优先级排序:
   - P0: 必须修改 (影响证明正确性)
   - P1: 强烈建议修改 (影响清晰度)
   - P2: 可选修改 (增强可读性)
4. 制定修改计划
5. 更新证明和论文
```
