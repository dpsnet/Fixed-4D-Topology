# 严格性检查清单 (Strictness Checklist)

**文档用途**: 系统化验证论文中每个声明的严格性等级  
**使用方式**: 每个Theorem/Conjecture必须通过所有相关检查项  
**验收标准**: 未通过检查项必须明确标注并制定修复计划

---

## L1 (Theorem) 验证清单

### 必要条件

- [ ] **符号完备性**: 所有符号在首次使用时已精确定义
- [ ] **假设明确**: 定理条件（if部分）无歧义
- [ ] **结论精确**: 定理结论（then部分）量化精确
- [ ] **无逻辑跳跃**: 证明中无"显然"/"众所周知"等未验证声明
- [ ] **边界情况**: 已考虑所有边界情况（如空集、零测度等）

### 充分条件

- [ ] **误差项控制**: 渐近公式中的O(·)有显式常数
- [ ] **一致性**: 参数依赖关系明确（如$C(\varepsilon_0, V_0)$）
- [ ] **必要性**: 已证明或讨论条件的必要性（反例构造）
- [ ] **Sharpness**: 已证明或讨论最优性（无法改进）

### 可追溯性

- [ ] **公理基础**: 所有推导可追溯至ZFC公理
- [ ] **文献引用**: 所有引用的结果有具体文献来源
- [ ] **原创性声明**: 明确标注哪些是原创，哪些是已知结果的新证明

---

## L3 (Conjecture) 验证清单

### 数值证据要求

- [ ] **样本量**: 明确报告测试案例数量 $N$
- [ ] **统计显著性**: 报告$R^2$、置信区间、$p$-值
- [ ] **误差分析**: 最大残差、标准误差
- [ ] **交叉验证**: 使用留一法或$k$-fold验证
- [ ] **反例搜索**: 报告已尝试但未找到反例的案例

### 理论支持

- [ ] **启发式论证**: 为什么这个猜想可能是真的
- [ ] **已知特例**: 已证明的特例（如对称情形）
- [ ] **障碍分析**: 证明该猜想的主要数学障碍
- [ ] **相关结果**: 文献中的类似猜想或定理

### 透明度

- [ ] **数据公开**: 数值数据可供验证
- [ ] **代码开源**: 计算代码公开（GitHub等）
- [ ] **可重复性**: 其他研究者可以复现结果

---

## L4 (Observation) 验证清单

- [ ] **数据来源**: 观察基于的具体数据或计算
- [ ] **模式描述**: 清晰描述观察到的模式
- [ ] **适用范围**: 明确说明观察的适用限制
- [ ] **未证明声明**: 明确标注"尚未证明"

## 术语使用规范

### 禁用术语（除非明确定义）

| 禁用术语 | 原因 | 替代方案 |
|---------|------|---------|
| "显然" | 主观判断 | 提供引用或简短证明 |
| "标准" | 模糊 | 指定具体文献 |
| "通常" | 不精确 | 给出量化条件 |
| "众所周知" | 可能不成立 | 提供引用 |
| "容易看出" | 主观 | 展开推导 |
| "直观的" | 非数学 | 删除或替换为形式论证 |

### 推荐术语

| 情境 | 推荐术语 | 示例 |
|-----|---------|------|
| 严格证明 | "我们证明..." | "我们证明$c(\delta)$由...给出" |
| 数值证据 | "我们观察到..." | "我们观察到相关性$R^2 = 0.97$" |
| 推测 | "我们猜想..." | "我们猜想系数$c_1$与...有关" |
| 开放问题 | "一个自然的问题是..." | "一个自然的问题是证明..." |
| 限制 | "在...条件下" | "在双曲性条件下..." |

## 交叉引用检查

确保所有引用正确：

- [ ] 每个Theorem有唯一的\label
- [ ] 每个\ref有对应的\label
- [ ] 文献引用格式统一
- [ ] 无"??"未解析引用
- [ ] 页码引用（如\pageref）准确

---

## 最终验收

论文通过严格性检查的标准：

1. **所有Theorem**: 通过L1检查清单
2. **所有Conjecture**: 通过L3检查清单，明确标注
3. **所有Observation**: 通过L4检查清单，明确标注
4. **无未标注的L3/L4**: 所有非严格声明已正确分类
5. **专家背书**: 至少1位领域专家预审通过
